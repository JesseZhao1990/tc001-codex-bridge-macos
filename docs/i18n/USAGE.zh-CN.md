# TC001 Codex Bridge for macOS - 完整使用说明

[← README](README.zh-CN.md) | [English usage guide](../USAGE.md)

## 安装与连接

1. 运行 `./run-tests.sh` 和 `./build.sh`。
2. 打开 `dist/TC001 Bridge.app`，允许蓝牙和本地网络权限。
3. 在设置中选择自动、Wi-Fi 或蓝牙。
4. Wi-Fi 模式填写设备 IP 或 `awtrix.local`；蓝牙模式等待 AWTRIX-BLE 设备连接。
5. 开启 Codex 自动监测并使用状态灯测试按钮确认显示。
6. 按需要配置五个 AWTRIX 内置页面开关。

## 显示含义

左侧 1x8 灯条表示 5 小时余额，右侧表示 7 天余额。设置中的两个额度开关支持只显示 5 小时、只显示 7 天或同时显示两种，且至少保留一种。只显示一种时画面固定在该额度并隐藏另一侧灯条；同时显示时 5H 显示 7 秒、7D 显示 3 秒。黄色为空闲、绿色为工作中、蓝色为等待确认、红色为异常。

## 顶部菜单与桌面小组件

- 打开“菜单栏显示额度”后，顶部状态图标旁会显示当前启用的 5H 和/或 7D 余额。
- 在 macOS 14 或更高版本的桌面空白处点按右键，选择“编辑小组件”，搜索“TC001 Bridge”，然后添加小号或中号组件。
- 这是由 WidgetKit 提供的原生桌面小组件；位置、尺寸、外观和移除操作均由 macOS 管理。
- 请让 TC001 Bridge 常驻菜单栏，小组件会读取最新的额度、模型状态和设备连接状态。应用未运行时，小组件会显示离线提示；点按即可启动应用。
- 5 小时与 7 天开关会同时控制 TC001、顶部菜单弹窗、菜单栏文字和桌面小组件。

## 故障排查

找不到蓝牙设备时检查固件版本 0.98-ble.4、系统蓝牙权限和旧配对记录；Wi-Fi 失败时确认 Mac 与 TC001 网络可达；没有额度时确认 Codex 已登录并等待刷新。小组件未出现在图库时，确认系统为 macOS 14 或更高版本，安装或更新后先启动一次 TC001 Bridge，再重新打开“编辑小组件”。

## 隐私与安全

项目没有遥测或自建服务器。应用只读取本地 Codex 状态，并向 TC001 发送渲染后的像素和页面开关。本地接口仅监听 127.0.0.1，并拒绝浏览器来源请求。
